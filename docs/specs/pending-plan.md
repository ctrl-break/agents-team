# Техническая спецификация: Детский трекер ежедневных дел (Kids Chore Tracker)

## Product Summary
Веб-приложение (SPA/PWA) для детей и родителей, позволяющее настраивать ежедневные дела, отслеживать их выполнение по дням недели, получать push-уведомления о задачах и формировать отчёты за выбранные периоды. Родители регистрируются через Google OAuth, добавляют email детей, дети авторизуются на своих мобильных устройствах и отмечают выполнение задач галочкой.

## Target Users
- **Дети (Children)**: 6–16 лет, мобильные устройства, отмечают выполнение дел, получают уведомления.
- **Родители (Parents)**: взрослые, управляют задачами детей, просматривают отчёты, контролируют выполнение.

## Core Features

1. **Google OAuth авторизация** — вход через Google-почту для родителей и детей.
2. **Связь родитель-ребёнок** — родитель добавляет email ребёнка, система создаёт семейную группу.
3. **Управление задачами (Chores)** — CRUD задач с привязкой к дням недели или типу «на неделе без фиксации дня».
4. **Отметка выполнения** — чекбокс выполнения задачи на сегодня/на определённый день.
5. **Push-уведомления** — гибкая настройка: время, день, частота напоминаний (Service Worker + Web Push API).
6. **Отчёты** — выбор периода, агрегация выполненных/пропущенных задач, визуализация (графики/таблицы).
7. **PWA** — установка на домашний экран устройства (manifest.json + Service Worker), оффлайн-поддержка отметки задач.
8. **Ролевая модель** — родитель видит всех своих детей, ребёнок видит только свои задачи.

## Assumptions
- Один ребёнок привязан к одному родителю (семейной группе). Допускается несколько детей у одного родителя.
- Email ребёнка уникален в системе. Родитель не может добавить email, уже привязанный к другому родителю.
- Задачи создаются с уровнем доступа: родитель может редактировать и удалять любые задачи своих детей, ребёнок — только свои.
- Push-уведомления работают только на устройствах с поддержкой Web Push (Chrome, Edge, Firefox на Android; Safari с ограничениями на iOS 16.4+).
- Часовой пояс — UTC, фронтенд приводит к локальному времени пользователя.
- Отчёт формируется по дате выполнения, а не по дате создания задачи.

## Technical Constraints
- **Фронтенд**: Angular 17+ (SPA), Angular Material или PrimeNG, PWA (Angular Service Worker), Angular Signals для state.
- **Бэкенд**: NestJS (Node.js 20+), TypeScript, TypeORM/Prisma.
- **База данных**: PostgreSQL 16.
- **Инфраструктура**: Docker Compose (3 службы: frontend, backend, postgres), Nginx для раздачи статики.
- **Авторизация**: Google OAuth 2.0, JWT access+refresh токены.
- **Уведомления**: Web Push Protocol (VAPID keys), хранение push-подписок в БД.

## Acceptance Criteria
- Родитель авторизуется через Google и видит панель управления детьми.
- При добавлении email ребёнка система отправляет приглашение на почту (или создаёт учётную запись для последующего входа ребёнка через Google).
- Ребёнок после входа видит только свои задачи.
- Задача имеет поля: название, описание, тип привязки (конкретный день/неделя), дни недели (пн-вс), активность, создатель.
- Отметка «выполнено» доступна только в день задачи (для дневных) или в любой день недели (для недельных), с возможностью снятия отметки.
- Отчёт отображает: % выполнения, список выполненных/пропущенных задач, график по дням.
- Push-уведомление приходит в настроенное время с текстом задачи.
- PWA устанавливается, работает оффлайн-кэширование последних данных.

## Backend Scope
- **Auth Module**: Google OAuth strategy (Passport), JWT генерация, guard для ролей `parent`/`child`.
- **Users Module**: профили пользователей, CRUD детей родителем, связь `parent_id -> children[]`.
- **Chores Module**: CRUD задач, поля: `title`, `description`, `schedule_type` (`daily_weekday` | `weekly`), `weekdays` (bitmask или jsonb), `is_active`, `created_by`, `assigned_to`.
- **Completions Module**: запись отметок `chore_id, user_id, date, completed_at`, проверка прав, уникальность на день.
- **Reports Module**: агрегация за период `start_date..end_date`, подсчёт статистики, возврат структуры для графиков.
- **Notifications Module**: хранение push-подписок, CRUD настроек уведомлений (`user_id, chore_id, time, days, enabled`), cron/очередь для отправки Web Push.
- **Database**: миграции через TypeORM/Prisma, seeds для тестовых данных.

## Frontend Scope
- **Страницы**: `/login` (Google OAuth callback), `/dashboard` (список дел на сегодня), `/chores` (управление задачами), `/kids` (управление детьми — только родитель), `/reports` (отчёты), `/settings` (профиль, уведомления).
- **Компоненты**: `ChoreCard`, `ChoreForm`, `WeekdayPicker`, `CompletionCheckbox`, `ReportChart`, `KidSelector`, `NotificationSettings`.
- **PWA**: Angular Service Worker (`ng add @angular/pwa`), manifest.json, иконки, оффлайн-страница.
- **State**: Signal Store или NgRx (облегчённый), кэширование через service worker.
- **UI**: Angular Material (темы, responsive, accessibility).
- **Push**: подписка через `SwPush`, передача subscription на бэкенд.

## QA Scope
- **Unit-тесты**: NestJS services, Angular компоненты (Jest/Jasmine).
- **E2E**: Playwright/Cypress — сценарии: регистрация родителя, добавление ребёнка, создание задачи, отметка выполнения, просмотр отчёта, настройка уведомлений.
- **PWA-тесты**: Lighthouse PWA audit, установка на домашний экран, оффлайн-режим.
- **Интеграционные**: API-контракты (OpenAPI/Swagger), Google OAuth flow.
- **Нагрузочные**: отложено на post-MVP.

## Architecture Concerns
- **Безопасность**: родитель не должен видеть чужих детей. Row-level security на уровне сервисов + проверка `family_id`.
- **Push-подписки**: срок действия подписок, обработка `expired` и `unsubscribed`, retry-логика при сбоях.
- **Оффлайн**: при отсутствии сети отметки выполнения кэшируются в IndexedDB и синхронизируются при восстановлении соединения (Background Sync).
- **Часовые пояса**: все даты на бэкенде в UTC, фронтенд конвертирует в локальный пояс пользователя для отображения и логики «день задачи».
- **Docker Compose**: `frontend` (Nginx + Angular build), `backend` (NestJS), `postgres` (с volume для данных), общая сеть, healthcheck'и.
